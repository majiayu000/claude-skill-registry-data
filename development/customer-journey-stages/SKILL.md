# 고객경험 단계 정의

## 목적

MVP 주제와 고객유형을 기반으로 현재 고객이 겪는 경험 단계를 정의하여, 이후 고객경험 조사와 여정 맵 작성의 기준을 제공합니다.

## 사용 시점

- 시장 조사가 완료된 후
- 고객경험 조사(관찰, 체험, 인터뷰) 시작 전
- 고객 여정 맵 작성 전
- 고객의 현재 경험 흐름을 구조화해야 할 때

## 필수 입력

- MVP 주제
- 고객유형 (JTBD 형식)
- 시장 조사 결과

## 고객경험 단계 정의 방법

### 1. 고객 행동 흐름 분석

고객이 문제를 인식하고 해결하기까지의 전체 흐름을 파악합니다:
- 문제 인식 시점
- 해결 방법 탐색
- 정보 수집
- 의사 결정
- 실행/사용
- 평가/관리

### 2. 경험 단계 도출

일반적인 경험 단계 패턴:

**정보/지식 관리 서비스**
```
니즈 인식 -> 방법 탐색 -> 수집 -> 분류/정리 -> 활용 -> 최적화/관리
```

**제품/서비스 구매**
```
니즈 발생 -> 정보 탐색 -> 비교/평가 -> 구매 결정 -> 사용 -> 재구매/추천
```

**일상 생활 서비스**
```
필요 인식 -> 옵션 탐색 -> 시도/체험 -> 습관화 -> 최적화 -> 유지/관리
```

**여행/경험 서비스**
```
영감/동기 -> 계획/준비 -> 예약 -> 경험 -> 기록/공유 -> 회상
```

### 3. 단계별 특징

각 단계는 다음을 포함해야 합니다:
- **명확성**: 단계명이 고객 행동을 명확히 설명
- **순차성**: 논리적인 흐름으로 연결
- **완결성**: 문제 인식부터 해결/관리까지 전체 커버
- **적절성**: 5-7개 단계로 구성 (너무 세분화하거나 추상적이지 않게)

## 작성 형식

### 출력 구조

```markdown
# 고객경험 단계 정의

## MVP 주제
{MVP 주제}

## 고객유형
{고객유형}

## 고객경험 단계
\```
{단계1} -> {단계2} -> {단계3} -> {단계4} -> {단계5} -> {단계6}
\```

## 단계별 설명

### {단계1}
- 주요 행동:
- 좋은 생각/느낌:
- 주요 Pain Points:

### {단계2}
- 주요 행동:
- 좋은 생각/느낌:
- 주요 Pain Points:

(각 단계별 반복)

## 단계 도출 근거
- {근거1}
- {근거2}
- {근거3}
```

## 사용 예시

**입력:**
```
MVP 주제: 생활 구독서비스 통합 관리
고객유형: 생활 구독서비스 이용자
```

**출력:**
```
니즈 인식 -> 방법 탐색 -> 수집 -> 분류/정리 -> 활용 -> 최적화/관리
```

**단계별 설명:**
- **니즈 인식**: 여러 구독 서비스 관리의 어려움 인식
- **방법 탐색**: 구독 서비스 관리 방법 검색
- **수집**: 현재 사용 중인 구독 정보 모으기
- **분류/정리**: 구독 종류별, 비용별 정리
- **활용**: 통합 대시보드로 모니터링
- **최적화/관리**: 불필요한 구독 해지, 비용 최적화

## 도구 활용

### Sequential MCP 사용
고객 행동 패턴을 체계적으로 분석하고 경험 단계를 도출하기 위해 Sequential MCP를 활용하세요.

## 결과 파일

- **고객경험단계.md**: `define/고객경험단계.md`
- 이후 관찰, 체험, 인터뷰에서 이 단계를 기준으로 조사 수행
- User Journey Map 작성 시 X축으로 사용

## 주의사항

- 너무 세분화하지 말고 5-7개 단계로 간결하게
- 고객 관점의 행동 중심으로 작성 (기업 프로세스가 아님)
- 단계는 화살표(->)로 연결
- 이후 모든 고객경험 조사는 이 단계를 기준으로 수행됩니다
- Sequential MCP를 활용하여 체계적으로 분석하세요
