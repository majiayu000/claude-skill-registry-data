# 서비스 기획서 발표자료 작성

## 목적

비즈니스 모델을 기반으로 서비스 기획서 발표자료 스크립트를 작성합니다.

## 사용 시점

- 비즈니스 모델 완성 후 발표 자료 준비가 필요할 때
- gamma.app 등 프레젠테이션 도구로 변환하기 전

## 핵심 요청사항

비즈니스 모델을 기반으로 서비스 기획서 발표자료 스크립트를 작성합니다.

**요청사항**:
- 예시와 같이 각 장표별 요약 작성
- 각 장표의 핵심 메시지는 3개 이하로 요약
- 각 장표 사이를 '---'으로 분리
- Sequential MCP 이용

**참고자료**:
- `think/비즈니스모델.md`

**예시**:
- `reference/sample_서비스기획서스크립트.txt`

**결과형식**: 마크다운

**결과파일**: `think/서비스기획서스크립트.md`

## 발표자료 표준 구조 (10-15장)

1. 표지: 서비스명, 한 줄 설명, 타겟 고객
2. 문제 정의: 고객의 Pain Point, 현재 대안의 한계
3. 시장 기회: TAM/SAM/SOM, 시장 성장률
4. 고객: 타겟 고객, 페르소나, Jobs To Be Done
5. 솔루션: 핵심 기능 Top 3, 차별점
6. 고유 가치 제안: "[고객]을 위한 [해결책]"
7. 제품 데모: 핵심 사용자 플로우, 주요 화면
8. 비즈니스 모델: 수익 모델, 가격 전략, LTV
9. 고객 확보 전략: 마케팅 채널, 파트너십
10. 경쟁 우위: 차별화 요소, 진입 장벽
11. 핵심 지표: North Star Metric, AARRR
12. 로드맵: 단기/중기/장기 계획
13. 팀: 핵심 팀원, 전문성
14. 재무 계획: 3년 전망, 투자 계획
15. 마무리: 비전, 요청사항, 연락처

## 스크립트 작성 원칙

- **간결성**: 각 슬라이드당 핵심 메시지 3개 이하
- **스토리텔링**: 문제 → 솔루션 → 가치의 흐름
- **비주얼 중심**: 텍스트 최소화, 비주얼로 설명
- **데이터 기반**: 주장을 데이터로 뒷받침
- **슬라이드 구분**: `---`로 각 슬라이드 구분
