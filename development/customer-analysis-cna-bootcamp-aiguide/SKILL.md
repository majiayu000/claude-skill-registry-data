# 고객 분석

## 목적

JTBD(Jobs To Be Done) 프레임워크를 사용하여 타겟 고객 유형을 정의합니다.

## 사용 시점

- MVP 정의가 완료된 후
- 고객 세그먼트를 명확히 해야 할 때
- 고객의 핵심 job을 파악해야 할 때
- 사용자가 "고객 분석", "타겟 고객", "페르소나"를 언급할 때

## 필수 입력
- MVP 주제 또는 제품 컨셉

## 분석 프레임워크

다음 형식으로 타겟 고객을 정의합니다:

### 타겟 고객 유형 (JTBD 형식)
[사용자 세그먼트 + 달성하려는 job]

**예시:**
- 집밥을 먹고 싶은 음식 소비자
- 비용 효율적인 생활 구독을 원하는 구독 서비스 이용자
- 맞춤형 여행 일정을 계획하고 싶은 여행 계획자

### 사용자 세그먼트
특성과 함께 3-5개의 주요 사용자 그룹을 식별합니다:
- 세그먼트 명
- 인구통계학적 특성
- 행동 패턴
- 현재 상황

### Job to be Done
고객이 달성하고자 하는 목표와 동기를 정의합니다:
- 주요 job
- 관련 jobs
- 원하는 결과

### 고객 Pain Points
고객이 현재 겪고 있는 5-7가지 주요 어려움을 나열합니다:
- 현재 문제점
- 불만 사항
- 장애물

### 고객 기대와 욕구
고객이 원하는 결과와 이상적인 경험을 설명합니다:
- 기대하는 결과
- 원하는 경험
- 성공 기준

## 작성 형식

```markdown
# 고객 분석

## MVP 주제
{MVP 주제}

## 타겟 고객 유형 (JTBD)
{사용자 세그먼트} + {달성하려는 job}

## 사용자 세그먼트

### 세그먼트 1: {세그먼트명}
- **인구통계**:
- **행동 패턴**:
- **현재 상황**:

(3-5개 세그먼트 반복)

## Job to be Done

### 주요 Job
{고객이 달성하려는 주요 목표}

### 관련 Jobs
- {관련 job 1}
- {관련 job 2}

### 원하는 결과
- {결과 1}
- {결과 2}

## 고객 Pain Points

1. {pain point 1}
2. {pain point 2}
(5-7개 나열)

## 고객 기대와 욕구

### 기대하는 결과
- {기대 1}
- {기대 2}

### 원하는 경험
- {경험 1}
- {경험 2}

### 성공 기준
- {기준 1}
- {기준 2}
```

## 중요 가이드라인

- "job to be done"은 명확한 기획을 위해 구체적이어야 합니다
- "원하는 것" 또는 "서비스 이용 이유"를 명확히 표현하세요
- 솔루션이 아닌 고객 목표에 집중하세요
- 구체적이고 관찰 가능한 행동을 사용하세요
- 가능한 경우 증거로 가정을 검증하세요

## 도구 활용

### Sequential MCP 사용
복잡한 고객 세그먼트 분석이 필요할 때 Sequential MCP를 활용하여 체계적으로 고객 특성을 도출하세요.

## 결과 파일

- **고객분석.md**: `define/고객분석.md`

## 주의사항

- JTBD는 "무엇을 달성하려는가"에 집중 (기능이 아님)
- 고객 관점에서 작성 (기업 관점 X)
- 구체적이고 측정 가능한 표현 사용
- 3-5개의 명확한 세그먼트로 구분
- 각 세그먼트의 차별화된 특성 명시
