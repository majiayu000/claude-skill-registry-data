# BM Analyzer (비즈니스 모델 분석기)

## Overview

현재 레포지토리의 서비스를 다양한 BM 전문가 관점에서 종합적으로 분석하는 스킬입니다.

**핵심 기능:**
- **코드베이스 분석**: 서비스의 핵심 기능과 가치 제안 파악
- **멀티 에이전트 분석**: 다양한 BM 전문가 역할의 에이전트가 병렬 분석
- **수익화 전략 제안**: 현실적인 수익화 방안 도출
- **개선점 및 로드맵**: 구체적인 실행 계획 제시

**BM 전문가 역할:**

| 역할 | 분석 관점 |
|------|-----------|
| 수익 전략가 (Revenue Strategist) | 수익 모델, 가격 전략, 매출원 다각화 |
| 시장 분석가 (Market Analyst) | 시장 규모, 경쟁사 분석, 포지셔닝 |
| 사용자 가치 분석가 (User Value Analyst) | 고객 세그먼트, 가치 제안, 사용자 여정 |
| 성장 전략가 (Growth Strategist) | 성장 채널, 바이럴 전략, 확장 가능성 |

## When to Use

이 스킬은 다음 상황에서 활성화됩니다:

**명시적 요청:**
- "BM 분석해줘"
- "비즈니스 모델 분석해줘"
- "수익화 분석해줘"
- "이 서비스로 어떻게 돈을 벌 수 있을지 분석해줘"
- "사업성 분석해줘"
- "monetization 분석"

**적합한 상황:**
- 새로운 프로젝트의 수익화 전략 수립
- 기존 서비스의 BM 개선
- 투자 유치를 위한 사업성 검토
- 사이드 프로젝트 상업화 검토

## Workflow

### 전체 분석 프로세스

```
┌─────────────────────────────────────────────────────────────┐
│ Phase 1: 코드베이스 탐색                                     │
│   ├─ README.md, package.json 등 핵심 문서 분석               │
│   ├─ 주요 기능 및 기술 스택 파악                             │
│   └─ 서비스 개요 정리                                        │
├─────────────────────────────────────────────────────────────┤
│ Phase 2: BM 전문가 병렬 분석 (planning-agents 활용)          │
│   ├─ 수익 전략가 분석                                        │
│   ├─ 시장 분석가 분석                                        │
│   ├─ 사용자 가치 분석가 분석                                 │
│   └─ 성장 전략가 분석                                        │
├─────────────────────────────────────────────────────────────┤
│ Phase 3: 개별 분석 결과 종합                                 │
│   ├─ 각 전문가 인사이트 정리                                 │
│   └─ 공통점/차별점 분석                                      │
├─────────────────────────────────────────────────────────────┤
│ Phase 4: 통합 BM 보고서 생성                                 │
│   ├─ 현재 상태 진단                                          │
│   ├─ 수익화 전략 권장안                                      │
│   ├─ 필요 개선사항                                           │
│   └─ 실행 로드맵                                             │
└─────────────────────────────────────────────────────────────┘
```

### Step 1: 코드베이스 탐색

서비스의 핵심 정보를 파악합니다:

```bash
# 분석 대상 파일들
- README.md            # 서비스 개요
- package.json         # 기술 스택, 의존성
- docs/               # 문서
- src/                # 핵심 기능
- API 엔드포인트       # 제공 기능
```

**추출 정보:**
- 서비스명 및 설명
- 핵심 기능 목록
- 기술 스택
- 현재 사용자/고객 정보 (있는 경우)
- 기존 수익 모델 (있는 경우)

### Step 2: BM 전문가 에이전트 분배

4명의 BM 전문가 역할을 Claude와 Codex에 랜덤 분배합니다:

```python
experts = [
    {"role": "revenue_strategist", "name": "수익 전략가"},
    {"role": "market_analyst", "name": "시장 분석가"},
    {"role": "user_value_analyst", "name": "사용자 가치 분석가"},
    {"role": "growth_strategist", "name": "성장 전략가"}
]

# 각 전문가에게 Claude 또는 Codex 할당
for expert in experts:
    expert["agent"] = random.choice(["claude", "codex"])
```

### Step 3: 병렬 BM 분석 실행

각 전문가에게 해당 역할의 관점에서 분석을 요청합니다.

**수익 전략가 분석 항목:**
- 가능한 수익 모델 (구독, 거래 수수료, 광고, 프리미엄 등)
- 가격 전략 제안
- 매출원 다각화 방안
- 수익성 예측

**시장 분석가 분석 항목:**
- TAM/SAM/SOM 추정
- 유사 서비스 및 경쟁사 분석
- 차별화 포인트
- 시장 진입 장벽

**사용자 가치 분석가 분석 항목:**
- 핵심 고객 세그먼트
- 고객 페인포인트 및 니즈
- 가치 제안(Value Proposition)
- 고객 획득 비용(CAC) 추정

**성장 전략가 분석 항목:**
- 성장 채널 제안
- 바이럴 계수 가능성
- 네트워크 효과 분석
- 확장성 평가

### Step 4: 개별 결과 출력

각 전문가의 분석 결과를 개별적으로 표시합니다:

```markdown
---
## 수익 전략가 (Claude) 분석 결과

[분석 내용]

---
## 시장 분석가 (Codex) 분석 결과

[분석 내용]

---
```

모든 분석을 종합하여 최종 보고서를 작성합니다:


## Configuration

### 기본 설정

```yaml
defaults:
  expert_count: 4              # 기본 전문가 수
  timeout: 180                 # 각 분석 타임아웃 (초)
  output_dir: "./bm_analysis"  # 결과 저장 위치

experts:
  - role: revenue_strategist
    weight: 1.2                # 수익 전략 가중치
  - role: market_analyst
    weight: 1.0
  - role: user_value_analyst
    weight: 1.0
  - role: growth_strategist
    weight: 1.0

codex:
  model: "o4-mini"

claude:
  use_current_session: true
```

## Best Practices

**DO:**
- 분석 전 README와 핵심 문서를 충분히 읽기
- 각 전문가 분석 결과를 비판적으로 검토
- 실현 가능한 수익화 방안에 집중
- 경쟁사 분석 결과를 실제 시장과 교차 검증
- 권장 액션에 구체적인 다음 단계 포함

**DON'T:**
- 기술적 우수성만으로 BM 가능성 판단
- 비현실적인 시장 규모 가정
- 경쟁사 분석 없이 차별화 주장
- MVP 없이 복잡한 수익 모델 설계
- 고객 검증 없이 가격 책정

## Troubleshooting

### 코드베이스 정보 부족

```
문제: README가 없거나 정보가 불충분함

해결:
1. 사용자에게 서비스 개요 직접 입력 요청
2. 코드 구조에서 기능 역추론
3. package.json 등 메타 파일 분석
```

### 전문가 분석 불일치

```
문제: 전문가들의 분석이 크게 상충됨

해결:
1. 상충 포인트 명시적으로 표시
2. 추가 분석 또는 사용자 판단 요청
3. 각 시나리오별 결과 병렬 제시
```

### 시장 정보 한계

```
문제: 실시간 시장 데이터 없음

해결:
1. 일반적인 업계 벤치마크 활용
2. 추정치임을 명확히 표기
3. 추가 조사 필요 항목 리스트업
```
