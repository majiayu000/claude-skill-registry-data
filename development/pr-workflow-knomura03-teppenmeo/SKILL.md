# PR運用

## 目的
- main直pushを避け、PR作成までを確実に行う

## 手順
1. 既存PRの未マージ有無を確認する
2. mainを最新化する
3. 作業ブランチを作成する
4. 変更を小さくコミットする
5. `pnpm lint` `pnpm test` `pnpm typecheck` を実行し結果を記録する
6. pushしてPRを作成する
7. PR本文に必須項目を記載する

## PR本文の必須項目
- 変更点要約
- 動作確認手順（クリック順）
- 実行コマンド結果（lint/test/typecheck）
- 影響範囲とロールバック観点
- 注意点（環境変数/モック）

## 事前チェック
- `git diff` `git status` を確認する
- `.env.local` や鍵/トークンが含まれていないことを確認する
- 疑わしい文字列は `git grep` で検査する

## 最終報告
- PR URLを必ず記載する
