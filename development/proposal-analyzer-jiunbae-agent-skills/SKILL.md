# Proposal Analyzer (제안서 분석기)

## Overview

사업 제안서 및 RFP 문서를 분석하여 사업 진행 타당성을 평가하는 스킬입니다.

**핵심 기능:**
- **문서 분석**: PDF, Word, 마크다운 등 다양한 형식 지원
- **정보 추출**: 사업명, 예산, 기간, 기술 요구사항 자동 추출
- **적정성 평가**: 가격, 기한, 기술 스펙의 적정성 분석
- **의사결정 지원**: 참여 권장도 및 리스크 평가 보고서 생성

**분석 관점:**
- 가격 적정성 (업계 평균 대비)
- 기한 적정성 (요구사항 대비 기간)
- 기술 스펙 적정성 (구현 난이도, 인력 수준)
- 수익성 분석 (마진율 평가)

## When to Use

이 스킬은 다음 상황에서 활성화됩니다:

**명시적 요청:**
- "제안서 분석해줘"
- "RFP 검토해줘"
- "입찰 참여할만한지 봐줘"
- "사업 타당성 분석해줘"
- "이 프로젝트 수주해도 될까?"

**자동 활성화:**
- 제안서/RFP 문서 파일이 제공될 때
- 사업 예산, 기간, 요구사항이 포함된 문서 분석 요청 시

**적합한 문서 유형:**
- 정부/공공기관 RFP
- 민간 기업 제안 요청서
- 입찰 공고문
- 프로젝트 견적 요청서

## Workflow

### 분석 프로세스

```
┌─────────────────────────────────────────────────────────┐
│ 1. 문서 읽기                                             │
│    └─ PDF, Word, 마크다운 등 문서 로드                   │
├─────────────────────────────────────────────────────────┤
│ 2. 핵심 정보 추출                                        │
│    ├─ 기본 정보 (사업명, 발주처, 개요)                   │
│    ├─ 일정 정보 (마감일, 수행기간, 마일스톤)             │
│    ├─ 예산 정보 (총액, 구성)                             │
│    ├─ 기술 요구사항 (스택, 기능, 성능, 보안)             │
│    └─ 자격 요건 (참가자격, 인증, 경험)                   │
├─────────────────────────────────────────────────────────┤
│ 3. 적정성 분석                                           │
│    ├─ 가격 적정성 (업계 평균 대비)                       │
│    ├─ 기한 적정성 (요구사항 대비)                        │
│    └─ 기술 스펙 적정성 (난이도, 인력)                    │
├─────────────────────────────────────────────────────────┤
│ 4. 보고서 생성                                           │
│    ├─ 종합 평가 및 참여 권장도                           │
│    ├─ 장점/리스크 분석                                   │
│    └─ 의사결정 체크리스트                                │
└─────────────────────────────────────────────────────────┘
```

### Step 1: 문서 읽기

Read 도구를 사용하여 제공된 문서를 읽습니다.

```bash
# 지원 형식
- PDF (.pdf)
- Word (.docx, .doc)
- 마크다운 (.md)
- 텍스트 (.txt)
- 한글 (.hwp) - 텍스트 추출 제한적
```

### Step 2: 핵심 정보 추출

문서에서 다음 정보를 추출합니다:

#### 기본 정보
| 항목 | 추출 내용 |
|------|-----------|
| 사업명 | 프로젝트/사업 명칭 |
| 발주처 | 발주 기관/회사명 |
| 사업 개요 | 목적, 범위, 배경 |

#### 일정 정보
| 항목 | 추출 내용 |
|------|-----------|
| 제안서 제출 마감일 | 마감 날짜/시간 |
| 사업 수행 기간 | 시작일 ~ 종료일 |
| 주요 마일스톤 | 중간 산출물 일정 |

#### 예산 정보
| 항목 | 추출 내용 |
|------|-----------|
| 총 사업 예산 | 총액 (VAT 포함/별도) |
| 예산 구성 | 인건비, 장비, 기타 비율 |

#### 기술 요구사항
- 필수 기술 스택
- 기능 요구사항
- 성능 요구사항
- 보안 요구사항

#### 자격 요건
- 참가 자격 (기업 규모, 형태)
- 필수 인증/자격증
- 유사 사업 경험

### Step 3: 적정성 분석

#### 가격 적정성
- 업계 평균 대비 예산 수준
- 인건비 단가의 합리성 (MM당 금액)
- 숨은 비용 또는 리스크 요소

#### 기한 적정성
- 요구사항 대비 기간의 충분성
- 준비 기간의 적절성
- 일정 리스크 요소

#### 기술 스펙 적정성
- 기술적 구현 난이도
- 필요 인력 수준
- 기술 리스크 요소

## Best Practices

**DO:**
- 문서 전체를 꼼꼼히 읽고 숨은 요구사항 파악
- 예산과 기간의 현실성을 객관적으로 평가
- 자격 요건 충족 여부를 먼저 확인
- 유사 프로젝트 경험과 비교 분석
- 리스크 요소를 구체적으로 명시
- whoami 스킬로 팀 역량 확인 후 분석

**DON'T:**
- 예산만 보고 판단 (숨은 비용 간과)
- 기한을 낙관적으로 해석
- 기술 난이도 과소평가
- 경쟁사 분석 없이 참여 결정
- 레퍼런스 가치만으로 저수익 프로젝트 수주

## Troubleshooting

### PDF 텍스트 추출 실패

```
문제: 스캔된 PDF에서 텍스트 추출 안 됨

해결:
1. OCR 처리된 PDF 요청
2. 수동으로 주요 정보 입력 요청
```

### 예산 정보 누락

```
문제: RFP에 예산 정보가 명시되지 않음

해결:
1. 유사 프로젝트 예산 참고
2. 요구사항 기반 예산 역산 제시
```

### 모호한 요구사항

```
문제: 기술 요구사항이 불명확함

해결:
1. 불명확한 항목 목록화
2. 질의응답(Q&A) 활용 권장
3. 최악/최선 시나리오 모두 분석
```
