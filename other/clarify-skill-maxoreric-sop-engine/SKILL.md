# 澄清意图（结伴开发模式）

通过对话逐步澄清用户的模糊想法，形成清晰的目标和产出定义。
**核心：先讨论设计，再实施。**

## 流程

1. **倾听** - 让用户描述想法，不打断
2. **提问** - 针对模糊点提出具体问题（每次 1-2 个，不轰炸）
3. **确认** - 用用户的语言复述，确保理解一致
4. **记录** - 把确定的内容写入自包含文档

## 提问策略

### 关于目标
- 你想解决什么问题？
- 成功的标准是什么？
- 有没有参考案例？

### 关于产出
- 最终产出是什么形式？
- 谁会使用这个产出？
- 有没有格式要求？

### 关于约束
- 有什么限制条件？
- 时间要求？资源限制？

## 输出：自包含设计文档

在 `workspace/<项目名>-design.md` 中维护，文档结构：

```markdown
# 项目名

---

# Part A：交互原则

> 复制本文档到新会话后，AI 应按照此原则继续工作。

## A.1 结伴开发模式
- 人给出模糊想法 → AI 提问澄清 → AI 记录笔记
- 先讨论设计，再实施
- 记录原话，不假设

## A.2 AI 职责
- 维护 Part B 的内容
- 每次对话后更新文档
- 重要决策和用户确认后再记录

---

# Part B：设计笔记

## 1. 用户原话
> [用 `>` 引用格式记录用户原始表述]

## 2. 已确定的决策
- [明确标注的决策点]

## 3. 详细设计
[目标、产出、成功标准、约束等]

## 4. 待定问题
- [ ] [未决事项]

## 5. TODO / Work Log
- [ ] [待办事项]
- [执行记录]
```

## 原则

- **记录原话** — 用 `>` 引用格式保留用户原始表述
- **不假设** — 有疑问就问，不要猜
- **允许演化** — 目标在讨论中可以变化
- **定期输出** — 生成自包含文档供用户保存/跨会话使用
