# TikTok Ads Best Practices Skill

## Overview

TikTok広告の運用・最適化のベストプラクティス。予算スケーリング、クリエイティブ戦略、キャンペーン構造、停止判断を網羅。

**トリガー**: TikTok広告の分析、予算変更、クリエイティブ作成、パフォーマンス評価時に使用。

## 1. 予算スケーリングルール

### 増加ルール

| フェーズ | 最大増加率 | 待機時間 |
|---------|-----------|---------|
| 学習フェーズ中 | 40%/回 | 2日以上 |
| 学習フェーズ後 | 20%/回 | 2-3日 |
| **推奨（安全）** | **15-20%** | **2-3日** |

### 学習フェーズ

| 項目 | 詳細 |
|------|------|
| 完了条件 | **7日間で50コンバージョン** |
| 再学習トリガー | 予算を一気に3倍以上に変更 |
| 禁止事項 | 学習フェーズ完了前のスケーリング |

### 警告サイン（スケーリング禁止）

| サイン | 詳細 |
|--------|------|
| CPA急上昇 | 予算増加直後に30%以上上昇 |
| CTR継続低下 | 2日連続10%以上低下 |
| 50 CV未達 | 過去7日間で50コンバージョン未満 |
| CPA余裕なし | 損益分岐点の80%以上 |

### 最低予算

| レベル | Daily Budget |
|--------|-------------|
| キャンペーン | $50/日以上 |
| 広告グループ | $20/日以上 |
| 推奨 | Target CPA × 10倍 |

## 2. クリエイティブベストプラクティス

### 動画仕様

| 項目 | 推奨値 |
|------|--------|
| アスペクト比 | **9:16（縦型）** |
| 解像度 | **1080×1920** |
| 長さ | **9-15秒** |
| ビットレート | 516kbps以上 |

### Hook戦略（最初の3秒）

| ルール | 詳細 |
|--------|------|
| **71%のユーザーが最初の3秒で視聴継続を判断** | |
| 問題提起から始める | 「6年間、何も変われなかった」 |
| ロゴは後回し | 最初にロゴを出さない |
| 動きで注目を引く | 静止画スタートは避ける |

### クリエイティブ疲労

| 項目 | 詳細 |
|------|------|
| 疲労速度 | Metaの**4倍速い** |
| 更新頻度 | **週1回以上** |
| 刷新タイミング | CTRが初期値の70%以下 |
| Frequency閾値 | 2.2で新クリエイティブ投入 |

### UGC vs ポリッシュ

| 項目 | UGC | ポリッシュ |
|------|-----|-----------|
| パフォーマンス | **3倍** | ベースライン |
| エンゲージメント | **2.5倍** | ベースライン |
| ROI | **+70%** | ベースライン |
| 推奨 | **強く推奨** | 避ける |

### クリエイティブ数

| フェーズ | 数量 |
|---------|------|
| テスト期 | 広告グループあたり3-5個 |
| 初期テスト | 10-20バリエーション並行 |
| スケール期 | 15-20個 |

### CTA（App Install）

| 推奨 | 避ける |
|------|--------|
| "Download" | "Learn more" |
| "Play game" | 曖昧な表現 |
| 視覚的オーバーレイ付き | テキストのみ |

## 3. キャンペーン構造

### Smart+ vs Manual

| 項目 | Smart+ | Manual |
|------|--------|--------|
| CPA | 10-15%低い | ベースライン |
| 必要条件 | 週50+コンバージョン | なし |
| 予算 | CPA×20倍 | CPA×10倍 |
| 推奨 | スケール時 | テスト時 |

**推奨戦略**: Manualでテスト → 勝ちパターンを見つけたらSmart+でスケール

### CPI設定（App Install）

| 項目 | 推奨値 |
|------|--------|
| 目標CPI | ¥600前後（Anicca設定済み） |
| 日次予算 | 目標CPI × 10倍 = ¥6,000/日 |
| Smart+時 | 目標CPI × 20倍 = ¥12,000/日 |

### ターゲティング

| 戦略 | 推奨 |
|------|------|
| **Broad Targeting** | **強く推奨** |
| Interest設定 | なし OR 1-2つのみ |
| 理由 | クリエイティブが最重要、AIに任せる |

### 広告グループ構造

| 項目 | 推奨値 |
|------|--------|
| CBO使用時 | 3-5グループ（最大7） |
| クリエイティブ/グループ | 3-5個 |

## 4. 停止判断基準

### メトリクス閾値

| メトリクス | 停止基準 |
|-----------|---------|
| CTR | 1.0%未満が2日以上継続 |
| CVR | 0%が$30以上消費後 |
| CPA | 損益分岐点の150%以上 |
| 最小データ量 | 5,000インプレッション以上で判断 |

### ストップロスルール

```
支出 > $30 かつ コンバージョン < 1（同日）
→ 即座に一時停止
```

### クリエイティブ疲労サイン

| サイン | アクション |
|--------|-----------|
| CTRが初期値の70%以下 | 新クリエイティブに差し替え |
| 7-10日経過 | 刷新検討 |
| Frequency 2.2以上 | 新クリエイティブ投入 |

## 5. A/Bテスト

### 黄金ルール

```
一度に1変数のみテスト
```

### 予算配分

| テスト対象 | 予算配分 |
|-----------|---------|
| クリエイティブ | 80% |
| オーディエンス | 15% |
| その他 | 5% |

### テスト順序（Modular Testing）

```
1. Hook（最初の3秒）
2. Visual Style（UGC vs ポリッシュ）
3. Script（メッセージ内容）
4. CTA
```

## 7. Anicca固有の推奨

### ペルソナ適合Hook

| Hook例 | ターゲット心理 |
|--------|--------------|
| 「6年間、何も変われなかった私が…」 | 長期の自己嫌悪 |
| 「習慣アプリ10個全部挫折した」 | 度重なる失敗経験 |
| 「また3日坊主で終わった」 | 諦めモード |

### 避けるべきHook

| 避けるHook | 理由 |
|-----------|------|
| 「簡単に習慣化！」 | 信じない、警戒する |
| 「たった○日で！」 | 過去の失敗経験と矛盾 |
| 楽観的すぎる表現 | ペルソナに刺さらない |

### 現在のパフォーマンス（2026年2月）

| 指標 | 値 | 目標 |
|------|-----|------|
| CPA | ¥360 | ¥2,000以下 ✅ |
| CTR | 4.96% | 2%以上 ✅ |
| 週間CV | 60 | - |
| 学習フェーズ | 完了 | ✅ |

## 8. チェックリスト

### 新キャンペーン開始時

- [ ] Broad Targetingに設定
- [ ] Daily Budget = Target CPI × 10
- [ ] クリエイティブ3-5個/広告グループ
- [ ] UGCスタイルで作成
- [ ] Hook（最初の3秒）で問題提起

### スケーリング前

- [ ] 学習フェーズ完了（50 CV/7日）
- [ ] CPA が損益分岐点の70-80%以下
- [ ] クリエイティブストック1-2週間分あり
- [ ] 増加率は20%以下

### 週次レビュー

- [ ] CTR低下なし（2日連続10%以上低下がない）
- [ ] CPA安定（30%以上の急上昇なし）
- [ ] クリエイティブ疲労チェック（7-10日経過していないか）
- [ ] 新クリエイティブ準備
