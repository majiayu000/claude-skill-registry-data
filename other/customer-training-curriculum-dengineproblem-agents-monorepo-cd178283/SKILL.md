## Learning Objective Hierarchy (Bloom's Taxonomy)

```yaml
Level 1 - Remember:
  - Product terminology
  - Feature location
  - Basic navigation
  Verbs: define, list, recall, identify

Level 2 - Understand:
  - Workflow concepts
  - Integration points
  - Use case mapping
  Verbs: explain, describe, summarize

Level 3 - Apply:
  - Basic tasks
  - Standard configurations
  - Common workflows
  Verbs: execute, implement, use

Level 4 - Analyze:
  - Troubleshooting
  - Data interpretation
  - Root cause analysis
  Verbs: compare, diagnose, investigate

Level 5 - Evaluate:
  - Optimization strategies
  - ROI assessment
  - Best practice selection
  Verbs: assess, recommend, prioritize

Level 6 - Create:
  - Advanced configurations
  - Custom solutions
  - Process design
  Verbs: design, build, develop
```

```yaml

  - Screen recording + voiceover
  - Captions always
  - Chapter markers for navigation
```

## Assessment Framework

### Knowledge Validation Model

```yaml
Formative (During learning):
  Type: Post-module quizzes
  Questions: 3-5 per module
  Passing: 70%
  Purpose: Identify gaps, reinforce learning

Summative (End of path):
  Type: Practical project + exam
  Duration: 60-90 minutes
  Passing: 80%
  Purpose: Certify competency

Reinforcement (Post-training):
  Type: Micro-assessments
  Timing: 30, 60, 90 days
  Purpose: Long-term retention
```

### Question Types

```yaml
Knowledge recall:
  - Multiple choice
  - True/false
  - Matching
  Use for: Terminology, facts

Application:
  - Scenario-based
  - Task simulation
  - Problem-solving
  Use for: Skills validation

Analysis:
  - Case studies
  - Troubleshooting scenarios
  - Best practice selection
  Use for: Expert certification
```
