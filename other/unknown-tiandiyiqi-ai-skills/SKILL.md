# 销售线索研究助手

此技能通过分析您的产品/服务、了解您的理想客户画像并提供可操作的拓展策略，帮助您识别和鉴定业务的潜在潜在客户。

## 何时使用此技能

- 为您的产品/服务寻找潜在客户或客户
- 建立要联系的公司列表
- 识别销售拓展的目标账户
- 研究符合您理想客户画像的公司
- 准备业务开发活动

## 此技能的功能

1. **了解您的业务**：分析您的产品/服务、价值主张和目标市场
2. **识别目标公司**：根据以下内容找到符合您理想客户画像的公司：
   - 行业和部门
   - 公司规模和位置
   - 技术栈和工具
   - 成长阶段和融资
   - 您的产品解决的痛点
3. **优先排序潜在客户**：根据匹配度和相关性对公司进行排名
4. **提供联系策略**：建议如何用个性化消息接近每个潜在客户
5. **丰富数据**：收集关于决策者和公司背景的相关信息

## 如何使用

### 基本用法

只需描述您的产品/服务以及您正在寻找的内容：

```
我正在构建[产品描述]。在[位置/行业]找到10家会成为
这产品好潜在客户的公司。
```

### 使用您的代码库

为了获得更好的结果，从产品的源代码目录运行此功能：

```
看看我正在这个仓库中构建的内容，在[位置/行业]识别10家
会从该产品中受益最多的公司。
```

### 高级用法

为了更有针对性的研究：

```
我的产品：[描述]
理想客户画像：
- 行业：[行业]
- 公司规模：[规模范围]
- 位置：[位置]
- 当前痛点：[痛点]
- 他们使用的技术：[技术栈]

找到20个符合条件的潜在客户，并提供每个的联系策略。
```

## 使用说明

当用户请求潜在客户研究时：

1. **了解产品/服务**
   - 如果在代码目录中，分析代码库以了解产品
   - 询问关于价值主张的澄清问题
   - 识别关键功能和优势
   - 理解它解决了什么问题

2. **定义理想客户画像**
   - 确定目标行业和部门
   - 识别公司规模范围
   - 考虑地理偏好
   - 理解相关的痛点
   - 注意任何技术要求

3. **研究和识别潜在客户**
   - 搜索符合标准的公司
   - 寻找需求的信号（职位发布、技术栈、最近的新闻）
   - 考虑成长指标（融资、扩张、招聘）
   - 识别具有互补产品/服务的公司
   - 检查预算指标

4. **优先排序和评分**
   - 为每个潜在客户创建匹配度评分（1-10）
   - 考虑因素如：
     - 与理想客户画像的一致性
     - 立即需求的信号
     - 预算可用性
     - 竞争格局
     - 时机指标

5. **提供可操作的输出**

   对于每个潜在客户，提供：
   - **公司名称**和网站
   - **为什么他们是好的匹配**：基于他们业务的特定原因
   - **优先级评分**：1-10及解释
   - **决策者**：目标角色/职位（例如"工程副总裁"）
   - **联系策略**：个性化方法建议
   - **价值主张**：您的产品如何解决他们的具体问题
   - **对话开场白**：拓展中要提及的具体要点
   - **LinkedIn URL**：如果可用，便于连接

6. **格式化输出**

   以清晰、可扫描的格式呈现结果：

   ```markdown
   # 潜在客户研究结果
   
   ## 摘要
   - 找到的潜在客户总数：[X]
   - 高优先级（8-10）：[X]
   - 中优先级（5-7）：[X]
   - 平均匹配度：[X]
   
   ---
   
   ## 潜在客户1：[公司名称]
   
   **网站**：[URL]
   **优先级评分**：[X/10]
   **行业**：[行业]
   **规模**：[员工数量/收入范围]
   
   **为什么他们是好的匹配**：
   [2-3个基于他们业务的特定原因]
   
   **目标决策者**：[角色/职位]
   **LinkedIn**：[URL（如果有）]
   
   **对他们的价值主张**：
   [对这家公司的具体好处]
   
   **拓展策略**：
   [个性化方法——提及具体痛点、最近的公司新闻或相关背景]
   
   **对话开场白**：
   - [具体要点1]
   - [具体要点2]
   
   ---
   
   [对每个潜在客户重复]
   ```

7. **提供后续步骤**
   - 建议保存结果到CSV以导入CRM
   - 提供起草个性化拓展消息
   - 建议基于时机的优先排序
   - 建议对有希望的潜在客户进行后续研究

## 示例

### 示例1：来自Lenny's Newsletter

**用户**："我正在构建一个工具，可以屏蔽AI编码助手中的敏感数据。找到潜在客户。"

**输出**：创建优先排序的公司列表，这些公司：
- 使用AI编码助手（Copilot、Cursor等）
- 处理敏感数据（金融科技、医疗保健、法律）
- 在他们的GitHub仓库中有使用编码代理的证据
- 可能意外在代码中暴露了敏感数据
- 包括相关决策者的LinkedIn URL

### 示例2：本地企业

**用户**："我经营一家远程团队生产力咨询公司。找到10家最近远程的湾区的公司。"

**输出**：识别这些公司：
- 最近发布了远程职位
- 宣布远程优先政策
- 正在招聘分布式团队
- 显示远程工作挑战的迹象
   - 为每个提供个性化拓展策略

## 获得最佳结果的提示

- **具体**关于您的产品及其独特价值
- **从您的代码库运行**以获得自动上下文
- **提供背景**关于您的理想客户画像
- **指定约束**如行业、位置或公司规模
- **请求后续**对有希望的潜在客户进行更深入的洞察
