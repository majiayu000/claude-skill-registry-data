# Page Optimizer (CRO)

マーケティングページ（LP、HP、価格ページ、機能ページ、ブログ）を分析し、改善提案→実装まで支援するスキル。

## Quick Start

1. 対象LPのURL or HTMLを取得
2. 構造分析を実施
3. 10項目のCROチェックリストで評価
4. 改善提案を優先度別に整理
5. Before/After形式で実装支援

## Core Workflow

### Phase 1: Discovery（発見）

ユーザーに確認:
- ターゲットユーザー（ペルソナ）
- 主要コンバージョン目標（DL/登録/問合せ）
- 現在の課題認識
- 競合LP（あれば）

### Phase 2: Audit（監査）

10項目を5段階評価:

1. **Hero Section** - ヘッドライン/サブヘッド/CTA
2. **Social Proof** - 数値/証言/ロゴ
3. **Benefits vs Features** - So What?テスト
4. **Conversion Path** - 摩擦点の最小化
5. **Visual Hierarchy** - F/Zパターン
6. **Copy Clarity** - スキャナビリティ
7. **Trust Elements** - 安心感
8. **Mobile Optimization** - Core Web Vitals
9. **Message Match** - 流入元との一貫性
10. **Competitive Diff** - 独自ポジショニング

### Phase 3: Recommendations（提案）

3カテゴリで整理:

**🔴 Critical（今すぐ/1-2日）**
- コンバージョン直結の致命的問題

**🟡 Important（1週間以内）**
- UX/コピーの中程度問題

**🟢 Nice-to-have（今後）**
- A/Bテストで検証推奨

### Phase 4: Implementation（実装）

各提案に対して:
1. Before/After例（コード含む）
2. 実装手順
3. 成功指標（KPI）

## Copywriting Quick Reference

**AIDA**: Attention → Interest → Desire → Action
**PAS**: Problem → Agitation → Solution

### Headline Formula

```
How to [達成したいこと] without [避けたいこと]
```

### CTA Best Practice

- アクション動詞で始める
- 価値を含める（「無料で始める」not「送信」）
- 1ページ1つの主要CTA

## A/B Test Hypothesis Template

```
「[変更内容]を実装すると、
 [ターゲットユーザー]の[行動指標]が
 [期待改善率]%改善する。
 根拠: [フレームワーク/データ]」
```

## ページタイプ別フレームワーク

### ホームページ CRO

複数オーディエンス対応が必要：
- 明確なポジショニングステートメント（コールド訪問者向け）
- 最も一般的なコンバージョンアクションへの明確なパス
- 訪問者が自己選択できるナビゲーション
- 「購入準備完了」と「まだリサーチ中」の両方に対応

### ランディングページ CRO

単一目的ページ：
- 流入元とのメッセージマッチ
- 単一CTA（可能ならナビゲーション削除）
- 1ページで完結する説得ロジック
- 本物の緊急性/希少性のみ使用

### 価格ページ CRO

高意図訪問者：
- 明確なプラン比較
- 推奨プランの表示
- 機能の明確化（含まれる/含まれない）
- 「どのプランが自分に合うか」不安への対応
- 価格→チェックアウトへの明確なパス

### 機能ページ CRO

詳細を調査中の訪問者：
- 機能→ベネフィットの接続
- ユースケースと例示
- 代替案との比較
- 試用/購入への明確なCTA

### ブログ記事 CRO

コンテンツ→コンバージョン：
- 記事トピックに関連するコンテキストCTA
- 記事主題に関連するリードマグネット
- 自然な停止ポイントでのインラインCTA
- 離脱意図をバックアップに

## 実験アイデア集

### ホームページ実験

**ヒーローセクション**
- ヘッドライン変更（具体的 vs 抽象的、ベネフィット vs 機能）
- サブヘッドラインの追加/改善
- ファーストビューCTAの有無
- ヒーロービジュアル（スクショ vs GIF vs イラスト vs 動画）
- CTAボタンカラー/コントラスト
- CTAテキスト（「無料トライアル」vs「始める」vs「デモを見る」）

**信頼・社会的証明**
- 顧客ロゴの配置（ヒーロー vs ビロウフォールド）
- ヒーローセクションでのケーススタディ/証言
- トラストバッジ（セキュリティ、コンプライアンス、受賞）

### 価格ページ実験

**価格表示**
- 年額割引の強調 vs 月額のみ vs 両方表示
- 価格ポイント（$99 vs $100 vs $97）
- 「最も人気」「おすすめ」バッジの追加
- 表示ティア数（3 vs 4 vs 2）
- プライスアンカリングの活用

**異論への対応**
- 価格FAQセクションの追加
- ROI計算機の追加
- 返金保証の目立つ表示

### デモリクエストページ実験

**フォーム最適化**
- フォームフィールド数の削減
- マルチステップ vs シングルステップ
- フォーム配置（ファーストビュー vs コンテンツ後）

### ブログ/リソースページ実験

**コンテンツCTA**
- フローティング/固定CTA
- インラインCTA vs 記事末尾のみ
- 関連リソースの記事末尾追加
