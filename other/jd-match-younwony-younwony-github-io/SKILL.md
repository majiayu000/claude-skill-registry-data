# JD 맞춤형 지원서 작성

> **원본 데이터**: `docs/career/my_career_data.md` (SSOT)
> **기본 경력기술서**: `docs/career/career_portfolio.md` (프로젝트 기간, AI Agent 등 참조)
> **출력 위치**: `docs/career/formats/by-jd/{company}/` 또는 `private/applications/{company}/`

---

## 핵심 원칙 ⭐

### 1. 시간순 정렬 (최신순)

> **모든 프로젝트는 시작일 기준 최신순으로 정렬**

```markdown
# 정렬 기준
1. 진행 중 프로젝트: 시작일 최신순
2. 완료된 프로젝트: 시작일 최신순
3. 동일 시작일: 중요도 순

# 예시 (올바른 순서)
- 인플루언서 데이터 플랫폼: 2024.01 - 현재
- 품절률 감소: 2023.06 - 2023.09
- 메인 페이지 큐레이션: 2022.08 - 2023.02
- ChatOps: 2022.06 - 2023.03
- Dynamic Pricing: 2022.03 - 2023.06
```

### 2. 기본 경력기술서 참조 (SSOT)

> **기본 경력기술서의 내용을 누락하지 않도록 반드시 참조**

```markdown
**필수 참조 항목:**
1. 프로젝트 기간 (정확한 시작/종료일)
2. AI Agent 활용 (Claude Code, MCP) - 개발 문화 개선 프로젝트
3. 정량적 성과 수치
4. Tech Decision 내용
```

### 3. 인재상 키워드 활용

> **회사의 인재상/비전을 지원서 전체에 자연스럽게 녹여내기**

---

## 개요

JD(채용공고)를 분석하여 맞춤형 지원서를 작성합니다.

### 출력 파일

```
docs/career/formats/by-jd/{company}/
├── COMPANY_RESEARCH.md # 회사 조사 결과 (비전, 성장, 인재상) ⭐ NEW
├── JD_ANALYSIS.md      # JD 분석 및 매칭 포인트
├── MOTIVATION.md       # 지원 동기 (회사 조사 기반) ⭐ NEW
├── COVER_LETTER.md     # 자기소개서 (기업 가치 기반)
└── RESUME_JD.md        # JD 맞춤형 이력서
```

> **private/applications/{company}/** 폴더에 직접 생성하는 경우도 허용 (spec.md, resume.md 등)

## 실행 순서

### Step 1: 회사 조사 (필수) ⭐ NEW

> **핵심**: JD만 보고 지원서를 쓰지 않는다. 회사의 비전/방향성을 먼저 파악한다.

```markdown
**조사 항목:**
1. 회사 비전/미션 (공식 홈페이지, 뉴스)
2. 사업 영역 및 성장 지표 (매출, 성장률, 상장 계획)
3. 최근 동향 (인수합병, 신규 서비스, 조직 확장)
4. 기술 스택/개발 문화 (기술 블로그, 채용 페이지)
5. 핵심 과제 추론 (성장에 따른 기술적 도전)
```

**조사 도구:**
- 공식 홈페이지: 비전, 서비스, 회사소개, 인재상
- THE VC / 코드너리: 투자, 기술 스택
- 뉴스 검색: 최근 동향, 인수합병
- 잡플래닛/블라인드: 개발 문화, 내부 분위기
- 채용 페이지: 인재상, 핵심 가치, 복리후생

**비전 및 인재상 확인 (필수):**

> 회사의 비전과 인재상은 지원 동기의 핵심 근거가 된다

```markdown
**확인 항목:**
1. 미션/비전 (Mission/Vision)
   - 회사가 추구하는 방향
   - 장기적 목표

2. 핵심 가치 (Core Values)
   - 회사가 중요시하는 가치 (예: 혁신, 협업, 고객 중심)
   - 개발 문화와 연결

3. 인재상 (Talent Profile)
   - 원하는 인재의 특성
   - 업무 스타일, 성향

4. 조직 문화 (Culture)
   - 애자일, 수평적 문화, 자율출근 등
   - 협업 방식
```

**비전-경험 매칭 템플릿:**

```markdown
| 회사 비전/인재상 | 내 경험 연결 | 지원 동기 포인트 |
|-----------------|-------------|-----------------|
| 고성장 플랫폼 | 200만 데이터 플랫폼 | 대용량 시스템 경험 |
| 혁신/도전 | 오픈소스 전환, TDD 도입 | 새로운 기술 도전 |
| 효율성/자동화 | ChatOps Zero Ops | 업무 자동화 기여 |
| 협업/소통 | Cross-functional 리딩 | 유관 부서 협업 |
```

**조사 결과 정리 템플릿:**

```markdown
## 회사 조사 결과

### 1. 회사 개요
- **업종**: [B2B SaaS, 커머스, 핀테크 등]
- **주요 서비스**: [서비스명 및 설명]
- **규모**: [직원 수, 고객사 수, 데이터 규모]

### 2. 성장 지표
- **매출**: [연 매출, 성장률]
- **투자**: [시리즈, 누적 투자금]
- **상장 계획**: [IPO 예정 등]

### 3. 기술적 도전 추론
| 회사 상황 | 예상 기술 과제 | 내 경험 연결 |
|----------|--------------|-------------|
| 고성장 | 시스템 확장성, 성능 최적화 | TPS 1500% 향상 |
| M&A 통합 | 레거시 전환, 데이터 통합 | 오픈소스 전환 |
| 대규모 SKU | 대용량 데이터 처리 | 200만 데이터 플랫폼 |

### 4. 비전-경험 연결
- **회사 비전**: [비전 문구]
- **연결 가능 경험**: [내 경험과 연결]
```

> **핵심**: 회사 조사 결과를 바탕으로 구체적인 지원 동기 작성

**지원 동기 구조 (4단락):**

```markdown
### 1. 고성장 환경에서의 기술적 도전
- 회사 성장 지표 언급 (매출, 성장률, 상장 계획)
- 내 대용량/성능 최적화 경험 연결

### 2. 도메인 경험의 시너지
- 회사 사업 영역과 내 도메인 경험 연결
- 구체적 유사 과제 해결 사례
- 직접 기여 가능 포인트

### 3. 기술 부채 개선과 개발 문화
- JD에서 언급된 기술 과제와 내 경험 연결
- TDD, 코드 리뷰, 레거시 전환 등
- 조직 문화 기여

### 4. 협업과 애자일 경험
- 회사 문화(Scrum 등)와 내 협업 경험 연결
- Cross-functional 리딩 사례
```

- [ ] 정량적 성과가 포함되었는가?

## JD 분석 프레임워크

### 1. 필수 요건 매칭 테이블

```markdown
| 요건 | 매칭 | 내 경험 |
|------|------|---------|
| Java 5년 이상 | 충족/부분/부족 | 구체적 경험 |
```

### 2. Pain Point 추론

> 단순 키워드 매칭이 아닌, **회사가 겪고 있는 기술적 통증** 추론

```markdown
**JD 키워드 → Pain Point 추론:**
- "대규모 트래픽" → 현재 성능 이슈 또는 확장 계획
- "MSA 전환" → 레거시 모놀리식 시스템 존재
- "글로벌 확장" → 다국어/다지역 지원 필요
```

### 3. 기업 가치 매칭 (자기소개서용)

```markdown
**기업 가치 → 내 경험 매칭:**
- Transform (변화 주도) → ChatOps 도입, TDD 문화 정착
- Innovation (혁신) → AI 도구 활용, 새로운 아키텍처 도입
- Collaboration (협업) → Cross-functional 리딩, 팀 생산성 향상
```

## 자기소개서 작성 가이드

### 구조 (STAR 기반)

```markdown
**[상황]**: 당시 조직의 문제/과제
**[행동]**: 내가 주도한 해결 방법
**[결과]**: 정량적 성과 + 조직 변화
**[연결]**: 기업 가치와의 연결
```

### 톤

- 구체적인 숫자와 성과 포함
- "~했습니다" 보다 "~달성했습니다" 능동적 표현
- 기술 나열보다 **문제 → 해결 → 성과** 구조

### 분량

- 메인 스토리: 600~800자
- 추가 어필 포인트: 선택적

## 인재상/비전 키워드 활용 가이드 ⭐ NEW

> **핵심**: 회사의 인재상과 비전 키워드를 지원서 전체에 자연스럽게 녹여내기

### 1. 인재상 키워드 추출

```markdown
**예시:**
- 인재상: 창의, 혁신, 열정, 도전
- 미션/비전: 회사 공식 홈페이지에서 확인
- 주요 업무: JD에서 추출 (기술 전파, 코드 개선, 성능 업그레이드 등)
```

### 2. 키워드-경험 매핑 테이블

```markdown
| 회사 키워드 | 내 경험 매핑 | 지원서 활용 위치 |
|------------|-------------|-----------------|
| **창의** | 비즈니스 관점 기술 해결책 (큐레이션, Dynamic Pricing) | 지원 동기, Summary |
| **혁신** | TPS 1,500% 향상, Zero Ops, AI Agent 활용 | Summary highlights, JD 태그 |
| **열정** | TDD/코드 리뷰 문화 정착, 팀 코드 품질 향상 | 지원 동기 |
| **기술 전파** | 코드 리뷰 문화 주도, MCP 도입 | JD 매칭 역량 |
```

### 3. Summary Highlights에 인재상 태그 활용

### 4. 지원 동기 작성 스크립트

> **각 문단에서 활용한 키워드를 명시적으로 설명**

```markdown
**지원 동기 작성 시 사용 키워드 안내:**

### 1. 도메인 전문성
- **활용 키워드**: JD 업무 키워드 + 회사 사업 영역
- **연결 경험**: 유사 도메인 프로젝트 경험
- **포인트**: 즉시 기여 가능한 도메인 역량 강조

### 2. 미션/비전 연결 - 혁신과 창의
- **활용 키워드**: 회사 미션 문구 + 인재상 키워드
- **연결 경험**: 정량적 성과 (성능 향상, 자동화, 신기술 도입)
- **포인트**: 기존 한계를 기술로 돌파한 경험

### 3. 조직 기여 - 성장과 열정
- **활용 키워드**: 인재상 키워드 + JD 업무 키워드
- **연결 경험**: 문화 개선, 프로세스 정착 사례
- **포인트**: 개인 성과를 넘어 조직 성장 기여
```
