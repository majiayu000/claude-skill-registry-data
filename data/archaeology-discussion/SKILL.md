# 발굴조사보고서 고찰 자동 작성 스킬

## 🎯 핵심 기능

**"고찰 작성해줘" 한 번의 명령으로 처음부터 끝까지 자동 실행**

1. 📁 폴더1(완료된 발굴조사보고서) 자동 분석
2. 📄 폴더2(학술논문) 핵심 내용 추출
3. 📊 폴더3(주변 유사 유적 보고서) 비교 데이터 수집
4. ✍️ 문화재청 표준양식에 맞춰 고찰 완전 생성

## 📂 필수 폴더 구조

프로젝트 루트에 다음 구조로 폴더를 준비하세요:

```
project/
├── 1_main_report/              # 폴더1: 완료된 발굴조사보고서 (주 자료)
│   ├── report.pdf              # 또는 .docx, .hwp
│   ├── tables/                 # 표 파일들
│   │   ├── stratigraphy.csv
│   │   ├── features.csv
│   │   ├── artifacts.csv
│   │   └── dating.csv
│   ├── figures/                # 도면 목록
│   │   └── list.md
│   └── extracted/              # (자동 생성) 추출된 데이터
│
├── 2_papers/                   # 폴더2: 학술논문
│   ├── paper1.pdf
│   ├── paper2.pdf
│   └── ...
│
└── 3_comparison_sites/         # 폴더3: 주변 유사 유적 보고서
    ├── site_A_report.pdf
    ├── site_B_report.pdf
    └── ...
```

다음 키워드가 포함된 요청을 받으면 **즉시 이 스킬을 활성화**하고 자동 파이프라인을 실행합니다:

### 트리거 키워드
- "고찰 작성해줘" ✅
- "고찰 써줘" ✅
- "고찰 작성" ✅
- "discussion 작성" ✅
- "발굴조사보고서 고찰" ✅
- "문화재청 양식 고찰" ✅

### 실행 전 자동 확인
1. **폴더 존재 확인**: `1_main_report/` 폴더가 있는가?
2. **보고서 파일 확인**: PDF/DOCX/HWP 파일이 1개 이상 있는가?
3. **선택 폴더 확인**: `2_papers/`, `3_comparison_sites/` 폴더가 있는가? (선택)

### 실행 개시
위 조건이 만족되면 **사용자에게 추가 질문 없이** Phase 1부터 자동 실행을 시작합니다.

```
✓ 폴더 구조 확인 완료
✓ 1_main_report/ 발견: report.pdf
✓ 2_papers/ 발견: 3개 논문
✓ 3_comparison_sites/ 발견: 5개 보고서

[Phase 1] 자료 수집을 시작합니다...
```

## 🔄 자동 파이프라인 절차

### ⚡ 전체 플로우 (자동 실행)

```mermaid
사용자 입력: "고찰 작성해줘"
    ↓
[준비] 폴더 스캔 (Glob)
    ↓
[Phase 1] 자료 수집
  → 1_main_report/ 읽기 (Read)
  → 핵심 데이터 추출 (Grep)
  → 2_papers/ 읽기
  → 3_comparison_sites/ 읽기
    ↓
[Phase 2] 고찰 생성
  → 9개 섹션 작성
  → 인용 삽입
  → 용어 통일
    ↓
[Phase 3] 검증 및 출력
  → 인용 검증
  → discussion_final.md 생성
```

### Phase 1: 자료 수집 및 분석 (자동)

#### Step 1.1: 주 보고서 분석 (폴더1)

**자동 실행 도구 사용 예시:**

```bash
# 1. 보고서 파일 탐지 (Glob)
Glob: "1_main_report/**/*.pdf"
Glob: "1_main_report/**/*.docx"
Glob: "1_main_report/**/*.hwp"

# 2. 보고서 읽기 (Read)
Read: "1_main_report/OO유적_보고서.pdf"
→ 전체 텍스트 추출

# 3. 핵심 데이터 추출 (Grep)
Grep: pattern="유적\s*위치|소재지" in "1_main_report/*.pdf"
→ "경기도 AA시 BB구 CC동 123번지"

Grep: pattern="조사\s*면적|발굴\s*면적"
→ "5,200㎡"

Grep: pattern="조사\s*기간"
→ "2024년 3월 ~ 6월"

Grep: pattern="유구.*?호|F\d+" (유구 번호)
→ F1, F2, ..., F20 (20개 유구)

Grep: pattern="층.*?L\d+|제\d+층" (층위)
→ L1, L2, L3, L4

# 4. 표 데이터 읽기 (CSV가 있는 경우)
Glob: "1_main_report/tables/*.csv"
Read: "1_main_report/tables/features.csv"
Read: "1_main_report/tables/artifacts.csv"
Read: "1_main_report/tables/dating.csv"

# 5. 도면 목록 읽기
Read: "1_main_report/figures/list.md"
```

**추출 결과 예시:**
```
유적명: OO유적
위치: 경기도 AA시 BB구
면적: 5,200㎡
조사기간: 2024.03~06
유구: 주거지 8동, 수혈 12기
유물: 토기 847점, 석기 378점
연대: 1400-1300 cal BC (청동기시대 전기)
```

#### Step 1.2: 논문 분석 (폴더2)

**자동 실행 도구 사용 예시:**

```bash
# 1. 논문 파일 탐지 (Glob)
Glob: "2_papers/**/*.pdf"

# 2. 논문 읽기 (Read) - 각 논문별로
Read: "2_papers/김철수_2020_한강유역청동기시대취락연구.pdf"
Read: "2_papers/이영희_2018_이중구연토기편년.pdf"
Read: "2_papers/박민수_2019_석촉형식분류.pdf"

# 3. 관련 내용 추출 (Grep)
Grep: pattern="청동기시대|청동기 시대" in "2_papers/*.pdf"
Grep: pattern="한강|경기|중부지방" in "2_papers/*.pdf"
Grep: pattern="이중구연토기|무문토기" in "2_papers/*.pdf"
Grep: pattern="주거지|수혈주거" in "2_papers/*.pdf"
Grep: pattern="cal BC|BP|연대|편년" in "2_papers/*.pdf"

# 4. 저자·연도 추출 (파일명 또는 본문)
Grep: pattern="^\d{4}년|^\d{4}\.|\((\d{4})\)"
→ 저자명 + 연도 매칭
```

**추출 결과 예시:**
```
논문1: 김철수 2020, "한강유역 청동기시대 취락 분포와 성격"
  - 주제: 청동기 전기 취락 구조
  - 관련: 주거지 평면 형태, 공간 배치
  - 인용문: "한강 중류역 청동기시대 전기 취락은 원형 주거지 중심..."

논문2: 이영희 2018, "이중구연토기의 형식분류와 편년"
  - 주제: 토기 편년
  - 관련: 이중구연토기 형식, 시기 구분
  - 인용문: "구연 각도 120-140°는 전기 후반의 특징..."

논문3: 박민수 2019, "중부지방 석촉 형식 연구"
  - 주제: 석기 형식
  - 관련: 유경식 석촉, 제작 기법
```

#### Step 1.3: 비교 유적 분석 (폴더3)

**자동 실행 도구 사용 예시:**

```bash
# 1. 비교 유적 보고서 탐지 (Glob)
Glob: "3_comparison_sites/**/*.pdf"

# 2. 각 보고서 읽기 (Read)
Read: "3_comparison_sites/가평_대성리유적.pdf"
Read: "3_comparison_sites/양평_양수리유적.pdf"
Read: "3_comparison_sites/포천_자작리유적.pdf"

# 3. 핵심 정보 추출 (Grep)
# 유적 위치
Grep: pattern="유적\s*위치|소재지|location" in "3_comparison_sites/*.pdf"
→ "가평군 대성리", "양평군 양수리", ...

# 시대·시기
Grep: pattern="청동기|철기|삼국|조선|cal BC|BP"
→ "청동기시대 전기", "1400-1200 BC", ...

# 유구 정보
Grep: pattern="주거지.*?\d+동|수혈.*?\d+기"
→ "주거지 12동", "수혈 8기", ...

# 유물 정보
Grep: pattern="이중구연토기|무문토기|석촉|석도"
→ 유물 조합 파악

# 규모
Grep: pattern="면적.*?\d+.*?㎡|조사.*?면적"
→ "15,000㎡", "8,200㎡", ...

# 4. 거리 계산 (위치 좌표가 있는 경우)
# 본 유적과 비교 유적 간 거리 추정
```

**추출 결과 예시:**
```
비교 유적 1: 가평 대성리유적
  - 위치: 가평군 대성리 (OO유적에서 북쪽 8km)
  - 시기: 청동기 전기 (1400-1200 BC)
  - 유구: 주거지 12동 (평균 면적 16.2㎡)
  - 유물: 이중구연토기 중심, 석촉 68점
  - 특징: 원형 주거지, 밀집 배치

비교 유적 2: 양평 양수리유적
  - 위치: 양평군 양수리 (OO유적에서 동쪽 15km)
  - 시기: 청동기 전기~중기 (1500-1000 BC)
  - 유구: 주거지 18동 (평균 면적 14.5㎡)
  - 유물: 이중구연토기, 공렬문토기, 옥 제품 6점
  - 특징: 대형 취락, 지역 중심지로 추정

비교 유적 3: 포천 자작리유적
  - 위치: 포천시 자작리 (OO유적에서 북쪽 20km)
  - 시기: 청동기 전기 (1400-1200 BC)
  - 유구: 주거지 6동
  - 유물: 무문토기, 석기 소량
  - 특징: 소규모 취락

→ 비교표 자동 생성 (comparison_table.csv)
```

### Phase 2: 고찰 생성 (자동)

#### Step 2.1: 문화재청 표준양식 적용

**9개 섹션에 Phase 1 데이터 자동 삽입:**

```
① 조사경위 및 연구사 (500-800자)
   ← 폴더1: 유적 위치, 조사 배경, 면적, 기간
   ← 폴더2: 지역 선행연구 (논문 인용)
   예: "OO유적은 경기도 AA시에 위치하며, DD천 상류 충적평야에 입지한다.
        본 조사는 EE고속도로 건설에 앞서 2024년 3월부터 6월까지 5,200㎡를
        발굴하였다. 이 지역은 청동기시대 취락이 밀집하는 것으로 알려져 있으나
        (김철수 2020), 개별 취락 구조 연구는 미진하였다."

② 층서 및 퇴적환경 (800-1,200자)
   ← 폴더1: 층위 구분, 토색, 토성, 문화층
   ← 폴더1 표: stratigraphy.csv
   예: "층서는 4개 층으로 구분된다(도 3, 표 1). 제1층(L1)은 현대 경작토이며,
        제3층(L3)은 암갈색(10YR 3/4) 점질토로 청동기시대 유구가 조성된 생활면이다.
        층 내 숯·토기편이 고르게 분포하여 인위적 퇴적으로 판단된다."

③ 유구 현황 및 해석 (1,500-2,500자)
   ← 폴더1: 유구 목록, 유형, 규모
   ← 폴더1 표: features.csv
   예: "주거지는 총 8동(F1-F8)이 확인되었다(표 2). 평면 형태는 원형 5동,
        타원형 3동이며, 면적은 10.2-18.7㎡(평균 13.8㎡)이다(도 5).
        주거지 F1은 직경 4.8m, 면적 18.1㎡로 본 취락의 중심 주거로 판단된다."

④ 유물 조합 및 형식 분류 (1,200-2,000자)
   ← 폴더1: 유물 통계
   ← 폴더1 표: artifacts.csv
   ← 폴더2: 형식 분류 이론 (논문 인용)
   예: "출토 유물은 총 1,247점이며, 토기 68%, 석기 30%, 골각기 2%이다(표 5).
        이중구연토기(237점)가 주를 이루고, 구연 각도 120-140°는 청동기시대 전기
        후반의 특징이다(이영희 2018)."

⑤ 연대 및 편년 (800-1,200자)
   ← 폴더1: C14 연대 결과
   ← 폴더1 표: dating.csv
   ← 폴더2: 편년 연구 (논문 인용)
   예: "주거지 F1 노지 목탄의 AMS 연대는 3,120±40 BP(Beta-567890)로,
        보정연대 1σ 범위는 1430-1380 cal BC이다(표 6). 이는 한강 유역
        청동기시대 전기 후반 편년(김철수 2020)과 부합한다."

⑥ 공간구조 및 생활상 (800-1,200자)
   ← 폴더1: 유구 배치, 분포 패턴
   예: "주거지 8동은 조사구역 북쪽에 밀집하며 평균 간격 8.5m이다(도 18).
        주거지 남쪽 20m 지점에 수혈 12기가 열상 배치되어, 주거역과 폐기역이
        구분되었음을 알 수 있다."

⑦ 주변 유적과의 비교 (1,000-1,500자)
   ← 폴더3: 비교 유적 데이터
   예: "본 유적의 주거지 평면 형태(원형)는 가평 대성리유적(8km 북쪽),
        양평 양수리유적(15km 동쪽)과 일치하여, 동일 문화권으로 판단된다(표 8).
        다만 어망추 비율(18%)이 주변 유적(평균 5%)보다 높아, 하천 어로에 특화된
        양상을 보인다."

⑧ 문화사적 의의 및 특징 (800-1,200자)
   ← 폴더1+2+3 종합
   예: "본 유적은 한강 중류역 청동기시대 전기 취락망의 일부로, 하천 자원
        활용과 광역 교역이라는 독자성을 지녔다고 평가된다."

⑨ 맺음말 (500-800자)
   ← 전체 요약 + 한계
   예: "본 조사는 청동기시대 전기 취락 구조 연구에 중요한 자료를 제공하였다.
        다만 조사 면적 제약(5,200㎡)으로 취락 전체 범위는 파악하지 못하였으며,
        식물·동물유체 분석이 미실시되어 구체적 생계 전략은 불명확하다."
```

#### Step 2.2: 논증 구조 생성
```
자동 실행 내용:
각 섹션마다 다음 구조 적용:
1. 주제문: 해당 섹션의 핵심 주장
2. 근거 제시: Phase 1에서 추출한 데이터 인용
   - 폴더1 데이터 → 1차 증거 (표/도 인용)
   - 폴더2 논문 → 이론적 배경 (저자 연도)
   - 폴더3 비교유적 → 지역적 맥락 (유적명)
3. 해석 및 추론: 증거→주장 논리 연결
4. 한계 명시: 불확실성, 대안 가능성
5. 소결: 다음 섹션 연결
```

#### Step 2.3: 인용 및 교차참조
```
자동 실행 내용:
1. 폴더1 표·도 자동 인용:
   - "표 3에서 보듯이"
   - "도 12의 주거지 평면은"
   - "유구 F5는"
2. 폴더2 논문 인용:
   - "(김OO 2020)"
   - "이OO(2018)에 따르면"
3. 폴더3 유적 비교:
   - "인근 AA유적(5km 북쪽)과 비교하면"
   - "표 10은 주변 유적 비교 결과이다"
4. 인용 일관성 자동 검증
```

#### Step 2.4: 문체 및 용어 통일
```
자동 실행 내용:
1. 학술적 문어체 변환:
   - "~이다/였다" (확실) vs "~로 판단된다" (추정)
   - "~가능성이 있다" (약한 추정)
2. 표준 용어 적용 (terminology_glossary.md):
   - "움집" → "수혈주거지"
   - "돌칼" → "석도" 또는 "석검"
   - "화덕자리" → "노지"
3. 정량적 표현:
   - "많다" → "68% (847점 중 576점)"
   - "크다" → "18.5㎡로 평균(13.2㎡)보다 40% 넓다"
```

### Phase 3: 검증 및 출력 (자동)

```

   - 인용 빈도 불균형 경고

   - 동일 개념 다른 용어 사용 탐지
3. 논리 검증:
   - 주장 without 근거 탐지
   - 근거와 주장 불일치 경고
4. 분량 검증:
   - 전체 8,000-12,000자 목표
   - 섹션별 500-1,500자 권장
```

## 🎓 핵심 원칙 (문화재청 지침 기반)

### 1. 증거 우선순위
1순위: 1차 관찰 데이터 (폴더1 보고서)
2순위: 정량 데이터 (통계, 연대측정)
3순위: 비교 자료 (폴더3 주변유적)
4순위: 이론적 배경 (폴더2 논문)

### 2. 논증 구조
```
주장 → 근거(폴더1) → 비교(폴더3) → 이론(폴더2) → 한계 → 대안
```

### 3. 인용 원칙
- 폴더1 데이터: 표·도·유구 번호로 인용
- 폴더2 논문: (저자 연도) 형식
- 폴더3 유적: 유적명 + 거리 명시

### 4. 문체 원칙
- 문어체 한국어
- 3인칭 객관적 서술
- 추정 강도 표현: "~이다" > "~로 판단된다" > "~로 추정된다" > "~가능성이 있다"
