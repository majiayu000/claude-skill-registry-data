# 会议洞察分析器

此技能将您的会议记录转化为关于沟通模式的可操作洞察，帮助您成为更有效的沟通者和领导者。

## 何时使用此技能

- 分析多次会议中的沟通模式
- 获取关于领导和促进风格的反馈
- 识别您何时回避困难对话
- 了解您的说话习惯和填充词
- 跟踪沟通技能的随时间改进
- 准备绩效评估并提供具体示例
- 指导团队成员的沟通风格

## 此技能的功能

1. **模式识别**：识别跨会议的反复出现的行为，例如：
   - 回避冲突或间接沟通
   - 说话比例和轮流发言
   - 提问与陈述模式
   - 积极倾听指标
   - 决策方式

2. **沟通分析**：评估沟通效果：
   - 清晰度和直接性
   - 填充词和委婉语言的使用
   - 语气和情感模式
   - 会议控制和促进

3. **可操作反馈**：提供具体的、有时间戳的示例，包括：
   - 发生了什么
   - 为什么这很重要
   - 如何改进

4. **趋势跟踪**：分析多次会议时比较随时间变化的模式

## 如何使用

### 基本设置

1. 将您的会议记录下载到文件夹（例如`~/meetings/`）
2. 在Claude Code中导航到该文件夹
3. 询问您想要的分析

### 快速开始示例

```
分析此文件夹中的所有会议，告诉我什么时候我回避了冲突。
```

```
看看我过去一个月的会议，识别我的沟通模式。
```

```
比较这两个会议文件夹中的我的促进风格。
```

### 高级分析

```
分析此文件夹中的所有记录并：
1. 识别我何时打断他人
2. 计算我的说话比例
3. 找到我回避给予直接反馈的时刻
4. 跟踪我使用填充词的情况
5. 展示良好积极倾听的示例
```

## 使用说明

当用户请求会议分析时：

1. **发现可用数据**
   - 扫描文件夹中的记录文件（.txt、.md、.vtt、.srt、.docx）
   - 检查文件是否包含说话者标签和时间戳
   - 确认会议的日期范围
   - 识别记录中用户的姓名/标识符

2. **澄清分析目标**

   如果未指定，询问他们想了解什么：
   - 具体行为（回避冲突、打断、填充词）
   - 沟通效果（清晰度、直接性、倾听）
   - 会议促进技能
   - 说话模式和比例
   - 待改进的成长领域

3. **分析模式**

   对于每个请求的洞察：

   **回避冲突**：
   - 寻找委婉语言（"也许"、"有点"、"我认为"）
   - 间接短语而非直接请求
   - 当紧张出现时改变话题
   - 不承诺地同意（"是的，但是..."）
   - 不解决明显的问题

   **说话比例**：
   - 计算会议中说话所占的百分比
   - 统计打断次数（您打断他人和被他人打断）
   - 测量平均发言时长
   - 跟踪提问与陈述的比例

   **填充词**：
   - 统计"嗯"、"啊"、"比如"、"你知道"、"其实"等
   - 注意每分钟或每次发言的频率
   - 识别它们在哪些情况下增加（紧张、不确定）

   **积极倾听**：
   - 引用他人先前观点的提问
   - 复述或总结他人的想法
   - 在他人贡献的基础上继续
   - 提问澄清问题

   **领导力与促进**：
   - 决策方式（指令性vs协作性）
   - 如何处理分歧
   - 包容较安静的参与者
   - 时间管理和议程控制
   - 后续跟进和行动项清晰度

4. **提供具体示例**

   对于发现的每个模式，包括：

   ```markdown
   ### [模式名称]
   
   **发现**：模式的单句摘要
   
   **频率**：在Y次会议中出现了X次
   
   **示例**：
   
   1. **[会议名称/日期]** - [时间戳]
      
      **发生了什么**：
      > [记录中的实际引用]
      
      **为什么这很重要**：
      [影响或错失机会的解释]
      
      **更好的方法**：
      [具体的替代措辞或行为]
   
   [重复2-3个最强的示例]
   ```

5. **综合洞察**

   分析所有模式后，提供：

   ```markdown
   # 会议洞察摘要
   
   **分析期间**：[日期范围]
   **分析的会议数**：[X个会议]
   **总时长**：[X小时]
   
   ## 发现的的关键模式
   
   ### 1. [主要模式]
   - **观察到**：您看到了什么
   - **影响**：为什么这很重要
   - **建议**：如何改进
   
   ### 2. [第二个模式]
   [相同结构]
   
   ## 沟通优势
   
   1. [优势1及示例]
   2. [优势2及示例]
   3. [优势3及示例]
   
   ## 成长机会
   
   1. **[领域1]**：[具体、可操作的建议]
   2. **[领域2]**：[具体、可操作的建议]
   3. **[领域3]**：[具体、可操作的建议]
   
   ## 说话统计
   
   - 平均说话时间：会议的X%
   - 提问数：每次会议平均X个
   - 填充词：每分钟X个
   - 打断：每次会议给予X次/收到Y次
   
   ## 下一步行动
   
   [改进沟通的3-5个具体行动]
   ```

6. **提供后续选项**
   - 在未来的会议中跟踪这些相同指标
   - 深入研究特定会议或模式
   - 与行业基准进行比较
   - 创建个人沟通发展计划
   - 生成绩效评估摘要

## 示例

### 示例1：冲突回避分析（灵感来自Dan Shipper）

**用户**："我下载了我所有的会议记录并把它们放在一个文件夹里。告诉我所有我微妙地回避冲突的时刻。"

**输出**：
```markdown
# 冲突回避模式

在15次会议中发现了23次您使用间接沟通或回避直接处理紧张的情况。

## 模式：对关键反馈的委婉表达

**频率**：7次会议中出现了8次

**示例1：与Sarah的1对1** - 00:14:32

**发生了什么**：
> "所以，我在想......也许我们可以，比如，
> 考虑重新看一下时间表？我的意思是，如果你觉得
> 有道理的话。但一切都听你的！"

**为什么这很重要**：
您需要解决Sarah的项目落后于进度的问题，但委婉语言（"也许"、"可能"、"如果你觉得"）
以及最后的回避（"一切都听你的"）让她很容易忽略紧迫性。

**更好的方法**：
"Sarah，项目落后了两周进度。我们需要讨论什么阻碍了进展，
并今天创建一个新的时间表。"

[更多示例...]

## 建议

1. **在第一句话中直接命名问题**
2. **删除委婉词**，如"也许"、"有点"、"某种程度上"
3. **提出具体问题**，而不是暗示
4. **安排困难对话**，而不是随意提出
```

### 示例2：领导力促进

**用户**："分析我的团队会议，告诉我关于我的促进风格。"

**输出**：提供关于以下内容的洞察：
- 您说话vs团队成员多少（60% vs 40%）
- 您是提问还是陈述（3:1比例）
- 您如何处理分歧（倾向于太快解决）
- 谁说话最少以及您是否引导他们发言
- 良好和错失促进时刻的示例

### 示例3：个人发展跟踪

**用户**："比较我第一季度vs第二季度的会议，看看我的倾听技能是否有所提高。"

**输出**：创建比较分析，显示：
- 打断减少（每次会议8次→3次）
- 澄清问题增加（每次会议2次→7次）
- 在他人想法基础上改进的示例
- 显示差异的具体示例
- 剩余的成长领域

## 设置提示

### 获取会议记录

**来自Granala**（订阅Lenny的通讯免费获得）：
- Granala自动转录您的会议
- 导出记录到文件夹：[操作说明]
- 将Claude Code指向该文件夹

**来自Zoom**：
- 启用带转录的云录制
- 会议后下载VTT或SRT文件
- 存储在专用文件夹中

**来自Google Meet**：
- 使用Google Docs自动转录
- 将记录文档保存到文件夹
- 下载为.txt文件或授予Claude Code访问权限

**来自Fireflies.ai、Otter.ai等**：
- 批量导出记录
- 存储在本地文件夹
- 对该文件夹运行分析

### 最佳实践

1. **一致的命名**：使用`YYYY-MM-DD - 会议名称.txt`格式
2. **定期分析**：每月或每季度回顾以了解趋势
3. **具体查询**：一次询问一种行为以获得深度
4. **隐私**：将敏感会议数据保留在本地
5. **行动导向**：一次专注于一个改进领域
