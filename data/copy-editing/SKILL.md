# 文案编辑

通过聚焦编辑扫描系统化改进现有文案，同时保留核心信息。

## 核心思路

好编辑不是重写，而是增强。每次扫描聚焦一个维度，避免一次改所有而遗漏问题。

## 七次扫描框架

### 扫描 1：清晰度
能否在 5 秒内理解？检查：混淆句式、代词不清、行话、模糊、缺背景。常见问题：一句说太多、抽象而非具体、假设读者已知、要点埋没在修饰中。  
**流程**：快速读并标记不清处 → 不立即改，先记录 → 标记后推荐具体编辑 → 确认编辑保持原意。

### 扫描 2：声音与语调
是否一致？检查：正式/随意切换、品牌个性不一致、情绪突兀、用词不匹配品牌。  
**流程**：朗读听不一致 → 标记意外切换 → 建议平滑过渡 → 确保个性贯穿。

### 扫描 3：So What
每项声明是否回答「我为什么要在意」？检查：功能无利益、声明无后果、未连接到读者生活、缺「这意味着…」桥梁。  
**测试**：对每项声明问「所以呢？」。若未回答更深利益，需改进。  
**流程**：逐项问「所以呢？」→ 标记缺答案的 → 添加利益桥梁或更深含义 → 确保利益连接真实读者需求。

### 扫描 4：证明
每项声明是否有证据？检查：无根据声明、缺社会证明、断言无支撑、「最佳」或「领先」无证据。  
**证明类型**：带姓名与细节的证言、案例研究引用、统计数据、第三方验证、保证与风险逆转、客户 logo、评分。  
**流程**：识别需证明的声明 → 检查附近是否有证明 → 标记无支撑断言 → 建议添加证明或软化声明。

### 扫描 5：具体性
是否足够具体？检查：模糊语言（「改进」「增强」「优化」）、通用声明、感觉编造的数字、缺细节。  
**升级**：模糊→具体（「节省时间」→「每周节省 4 小时」）。  
**流程**：标记模糊词句 → 问「能否更具体？」→ 添加数字、时间或示例 → 删除无法具体化的内容（可能是填充）。

### 扫描 6：提升情绪
是否让读者有感觉？检查：平淡信息性语言、缺情绪触发、痛点提及但未感受、愿望陈述但未唤起。  
**情绪维度**：当前状态痛苦、替代方案挫败、错失恐惧、转变渴望、明智选择自豪、解决问题解脱。  
**流程**：读情绪影响 → 识别应共鸣的平淡部分 → 添加情绪质感同时保持真实 → 确保情绪服务于信息。

### 扫描 7：零风险
是否移除所有行动障碍？检查：CTA 附近摩擦、未回答异议、缺信任信号、下一步不清、隐藏成本或意外。  
**风险降低**：退款保证、免费试用、「无需信用卡」「随时取消」、CTA 附近社会证明、清晰期望、隐私保证。  
**流程**：聚焦 CTA 附近部分 → 列出可能犹豫的原因 → 检查文案是否处理每项担忧 → 按需添加风险逆转或信任信号。

**每次扫描后**：返回前序扫描确认未引入新问题。

## 快速检查

**词级**：删除「very/really/extremely」「just/actually/basically」「in order to」→「to」「that」（常不必要）「things/stuff」。替换「utilize」→「use」「implement」→「set up」「leverage」→「use」等。  
**句级**：一句一意、长短混合、前置重要信息、最多 3 个连词、通常不超过 25 字。  
**段级**：一段一主题、短段（网页 2～4 句）、强开头、段落间逻辑、留白便于扫描。

## 常见问题与修复

- **功能墙**：功能列表无「为什么重要」→ 每项后加「这意味着…」桥梁。  
- **公司腔**：用「人类会怎么说」替换。  
- **弱开头**：以公司历史或模糊陈述开头 → 以读者问题或期望结果开头。  
- **CTA 埋没**：请求在过多铺垫后或不清晰 → 使 CTA 明显、早出现、重复。  
- **无证明**：「客户爱我们」无证据 → 添加具体证言、数字或案例引用。  
- **通用声明**：「我们帮助业务增长」→ 具体说明谁、如何、多少。  
- **混合受众**：试图对所有人说话 → 选一个受众直接写。  
- **功能过载**：列出所有能力 → 聚焦 3～5 个对受众最重要的利益。
