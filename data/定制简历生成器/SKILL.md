# 定制简历生成器

## 何时使用此技能

- 申请特定职位
- 为不同行业或角色定制简历
- 突出相关经验以实现职业转型
- 优化简历以通过ATS（求职者追踪系统）
- 为不同求职申请创建多个简历版本
- 强调职位描述中提到的特定技能

## 此技能的功能

1. **分析职位描述**：从职位发布中提取关键要求、技能、资格和关键词
2. **识别优先级**：根据职位描述的语言和结构确定雇主最看重什么
3. **定制内容**：重新组织并强调相关经验、技能和成就
4. **优化关键词**：自然地将ATS友好的关键词融入整个简历
5. **专业格式**：创建适合各种格式的简洁、专业简历布局
6. **提供建议**：建议改进并突出需要解决的差距

## 如何使用

### 基本用法
提供职位描述和您的背景信息：

```
我正在申请这个职位：

[粘贴职位描述]

这是我的背景：
- 在TechCorp担任软件工程师5年
- 领导3人开发团队完成移动应用项目
- 精通Python、JavaScript、React
- 州立大学计算机科学学位
```

### 使用现有简历
上传或粘贴您当前的简历以及职位描述：

```
请为这个职位定制我的简历：

职位描述：
[粘贴职位描述]

我的当前简历：
[粘贴简历内容]
```

### 职业转型
当换行业或角色时：

```
我正在从市场营销转向产品管理。
这是我正在申请的职位：

[粘贴职位描述]

我的可迁移经验：
- 7年数字营销经验
- 领导跨职能团队
- 管理产品发布
- 数据分析和A/B测试
```

## 使用说明

当用户请求定制简历时：

### 1. 收集信息

**职位描述分析**：
- 如果未提供，请求完整的职位描述
- 询问公司名称和职位名称

**候选人背景**：
- 如果用户提供了现有简历，请将其作为基础
- 如果没有，请询问：
  - 工作经历（职位名称、公司、日期、职责）
  - 教育背景
  - 关键技能和技术能力
  - 显著成就和指标
  - 证书或奖项
  - 任何其他相关信息

### 2. 分析职位要求

提取并优先考虑：
- **必备资格**：经验年限、必备技能、教育背景
- **关键技能**：技术工具、方法论、能力
- **软技能**：沟通、领导力、团队协作
- **行业知识**：领域特定经验
- **关键词**：重复的术语、短语和用于ATS优化的流行语
- **公司价值观**：从职位描述中得出的文化契合度指标

创建思维地图：
- 优先级1：关键要求（交易破坏者）
- 优先级2：重要资格（强烈期望）
- 优先级3：最好具备的技能（加分项）

### 3. 将候选人经验映射到要求

对于每个职位要求：
- 从候选人背景中识别匹配的经验
- 如果没有直接匹配，找到可迁移的技能
- 记录需要解决或淡化的差距
- 识别需要突出的独特优势

### 4. 构建定制简历

**专业摘要**（3-4行）：
- 以目标领域/角色的经验年限开头
- 包含职位描述中前3-4项必备技能
- 如果相关，提及行业经验
- 突出独特的价值主张

**技术/核心技能部分**：
- 按与职位要求匹配的类别对技能进行分组
- 首先列出必备工具和技术
- 使用职位描述中的确切术语
- 只包括你能用经验证明的技能

**工作经验**：
- 对于每个角色，强调与职位要求一致的责任和成就
- 使用动作词：领导、开发、实施、优化、管理、创建、分析
- **量化成就**：包括数字、百分比、时间范围、规模
- 重新排序要点以优先考虑最相关的经验
- 自然地使用职位描述中的关键词
- 格式：**【动作词】+【做了什么】+【如何/为什么】+【结果/影响】**

**教育**：
- 列出与职位相关的学位、证书
- 如果是早期职业，添加相关课程
- 添加与职位要求匹配的证书

**可选部分**（如果适用）：
- 证书和执照
- 出版物或演讲活动
- 奖项和认可
- 志愿服务（如果与角色相关）
- 项目（尤其是技术角色）

### 5. 优化ATS（求职者追踪系统）

- 使用标准部分标题（专业经验、教育、技能）
- 自然地融入职位描述中的确切关键词
- 避免表格、图形、页眉/页脚或复杂格式
- 使用标准字体和项目符号
- 包括首字母缩写词和完整术语（例如，"SQL（结构化查询语言）"）
- 在诚实的前提下匹配职位名称术语

### 6. 格式化和呈现

**格式选项**：
- **Markdown**：简洁、可读、易于复制
- **纯文本**：ATS优化，安全适用于所有系统
- **Word/PDF提示**：提供格式指导

**简历结构指南**：
- 10年以下经验保持1页，10年以上经验保持2页
- 使用一致的格式和间距
- 确保联系信息突出
- 使用逆时间顺序（最近期优先）
- 保持干净、可扫描的布局，留有空白

### 7. 提供战略建议

呈现定制简历后，提供：

**优势分析**：
- 是什么让这位候选人具有竞争力
- 在求职信或面试中强调的独特资格

**差距分析**：
- 未完全满足的要求
- 解决差距的建议（课程、项目、重塑经验）

**面试准备技巧**：
- 与简历一致的谈话要点
- 基于职位要求准备的故事
- 展示契合度时要问的问题

**求职信钩子**：
- 建议2-3行求职信开场
- 详细阐述的关键成就

### 8. 迭代和完善

询问用户是否想要：
- 调整重点或语气
- 添加或删除部分
- 为不同角色生成替代版本
- 创建格式变体（传统与现代）
- 开发特定角色版本（如果申请多个相似职位）

### 9. 遵循的最佳实践

**做**：
- 诚实准确——永远不要编造经验
- 使用行业标准术语
- 用具体指标量化成就
- 为特定职位定制每份简历
- 校对语法和一致性
- 保持语言简洁有力

**不做**：
- 包含个人信息（年龄、婚姻状况、照片，除非被要求）
- 使用第一人称代词（我、我的）
- 包括参考（"可应要求提供"已过时）
- 如果职业生涯超过20年，列出每份工作（专注于相关、近期经验）
- 使用未经定制的通用模板
- 超过2页，除非是非常高级的职位

### 10. 特殊考虑

**职业转型者**：
- 使用功能性或混合简历格式
- 强调可迁移技能
- 在摘要中创建引人注目的叙述
- 专注于相关项目和课程

**应届毕业生**：
- 以教育背景开头
- 包括相关课程、项目、实习
- 强调学生组织中的领导力
- 如果GPA在3.5以上包括GPA

**高级管理人员**：
- 以执行摘要开头
- 专注于领导力和战略影响
- 包括董事会成员资格、演讲活动
- 强调收入增长、团队建设、愿景

**技术角色**：
- 显著包括技术技能部分
- 列出编程语言、框架、工具
- 包括GitHub、作品集或项目链接
- 提及方法论（Agile、Scrum等）

**创意角色**：
- 包括作品集链接
- 突出创意成就和活动
- 提及工具和软件熟练程度
- 考虑更具创意的格式（同时保持ATS兼容性）

## 获得最佳结果的提示

- **具体**：提供完整的职位描述和详细的背景信息
- **分享指标**：在描述经验时包括数字、百分比和可量化的成就
- **指示格式偏好**：如果您需要ATS优化、创意或传统格式，请让技能知道
- **提及约束**：分享任何具体要求（页数限制、要包含/排除的部分）
- **迭代**：不要犹豫要求修改或替代方法
- **多个申请**：为不同角色生成单独的定制版本

## 隐私说明

此技能处理您的个人和专业信息以生成定制简历。在提交之前始终审查输出以确保准确性和适当性。删除或修改您不希望与潜在雇主分享的任何信息。
