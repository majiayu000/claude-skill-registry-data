# 小红书自动发帖

## 概述

此技能通过AutoGLM的手机控制能力实现小红书APP的自动发帖功能。提供两种发帖类型的结构化工作流程：普通笔记（写想法）和长文笔记（写长文）。从内容创作到最终发布，全程指导AutoGLM完成发帖流程。

## 工作流程决策树

在开始任何发帖流程之前，首先与用户确认帖子类型：

1. **普通笔记（写想法）** - 以卡片样式展示的内容
   - "写想法"预览内容必须简洁（十字以内）
   - "写想法"内容会在首页大图展示给其他用户
   - 最终发布时需要输入完整的正文内容
   - 适合快速分享想法、更新或内容
   - 使用卡片模板进行视觉呈现

2. **长文笔记（写长文）** - 转换为图片的详细内容
   - 内容必须充实（至少100字）
   - 文字转换为图片以提供更好的阅读体验
   - 包含自动排版和布局选项
   - 需要添加与图片内容不同的正文摘要

**决策流程：**
- 询问用户："您想发布哪种类型的笔记？普通笔记（简短内容）还是长文（详细内容）？"
- 根据回答，执行下面相应的工作流程

## 工作流程一：发布普通笔记

当用户想要发布短内容作为普通笔记时使用此流程。

### 步骤1：打开小红书并进入发布界面

1. 确保手机上小红书APP已打开
2. 点击屏幕底部中央的红色加号按钮
3. 从选项中选择"写文字"

### 步骤2：输入"写想法"预览内容

1. 定位并点击"写想法"输入框
2. 输入简洁的预览内容
   - **关键约束**：内容必须简洁，十字以内
   - **重要说明**：这个内容会在首页大图展示给其他用户
   - 应该提炼核心观点或吸引眼球的短句
   - 这不是最终的正文内容，只是预览
   - 完整的正文内容将在后续步骤中输入

### 步骤3：选择卡片风格

1. 输入预览内容后，点击"下一步"
2. 浏览可用的卡片样式模板
3. 选择卡片风格
   - **可以询问用户**：想要哪一种卡片风格
   - 考虑：配色方案、字体样式、布局设计
   - 建议选择可读性好、与内容调性匹配的卡片
4. 选择完成后再次点击"下一步"

### 步骤4：添加标题、输入正文并发布

1. 点击"添加标题"
2. 输入一个总结内容的吸引人标题
   - 保持简洁且引人注目
   - 使用相关关键词以提高可发现性

3. **重新输入完整的正文内容**
   - **关键说明**：正文内容默认显示的是"写想法"栏的内容（只有十字）
   - **必须操作**：使用真正需要发布的完整内容覆盖当前的正文内容
   - 定位正文输入框（通常在标题下方）
   - 删除或覆盖默认的短内容
   - 输入完整的、用户想要发布的正文内容
   - 正文可以比"写想法"预览内容更详细、更长

4. 完成标题和正文后，点击"发布笔记"
5. 等待确认帖子已成功发布

## 工作流程二：发布长文笔记

当用户想要发布详细、充实的内容作为长文时使用此流程。

### 步骤1：打开小红书并进入发布界面

1. 确保手机上小红书APP已打开
2. 点击屏幕底部中央的红色加号按钮
3. 从选项中选择"写文字"

### 步骤2：进入长文编辑器

1. 定位并点击"写长文"按钮
2. 这将打开长文编辑界面

### 步骤3：输入标题和内容

**重要提示**：标题和正文是两个独立的输入框，不要将正文内容接着标题输入！

1. 在标题栏输入文章标题
   - 标题应具有描述性和吸引力
   - 反映文章的主要话题
   - **输入完标题后，不要继续输入正文内容**

2. 在标题下方找到正文输入框并输入主要内容
   - **关键识别**：正文输入框显示"粘贴到这里或输入文字，内容将自动保存"
   - **操作方式**：点击该占位符文字偏下方的位置（而非直接点击文字），确保光标在正文区域
   - **点击位置说明**：点击占位符文字下方约50-100像素的位置，避免误触到标题输入框
   - **关键约束**：内容必须至少100字
   - 此内容将被转换为图片以提供更好的阅读体验
   - 内容应该详细且全面
   - 使用适当的格式（段落、换行）提高可读性

### 步骤4：应用自动排版

1. 点击右上角的"一键排版"
2. 等待系统处理并生成格式化的布局选项
3. 浏览可用的排版模板
4. 选择一个能增强内容可读性的美观布局
5. 选择布局后点击"下一步"

### 步骤5：添加正文摘要并发布

1. 定位"添加正文"输入框
2. 输入摘要或补充文字
   - **关键约束**：此文字不能与图片内容重复
   - 图片内容更详细；此摘要提供背景或重点
   - 用户通常更倾向于阅读图片中的内容
   - 使用此空间：预告内容或提供额外背景
   - **话题标签**：仅在用户明确要求时添加（如用户指定了需要添加的标签）

3. 完成摘要后，点击"发布笔记"
4. 等待确认文章已成功发布

## AutoGLM操作关键指南

使用AutoGLM执行这些工作流程时，遵循以下原则：

### 内容约束（关键）
- **普通笔记"写想法"预览**：十字以内 - 严格执行
- **普通笔记正文**：在最终发布界面输入完整内容，覆盖默认的"写想法"短内容
- **长文笔记**：至少100字 - 执行前验证
- **内容分离**：图片内容和正文摘要必须不同且不重复

### UI导航提示
- 在继续操作前，始终等待UI元素完全加载
- 寻找特定的按钮文字匹配："写文字"、"写想法"、"写长文"、"下一步"、"添加标题"、"发布笔记"
- 红色加号按钮通常位于底部导航栏中央
- 如果导航失败，重试或请用户手动验证当前屏幕
- **长文输入特别注意**：
  - 标题输入框和正文输入框是两个独立的UI元素
  - 输入标题后，必须定位到显示"粘贴到这里或输入文字，内容将自动保存"的正文输入框
  - **点击位置**：点击该占位符文字偏下方的位置（下方约50-100像素），而非直接点击文字本身
  - 这样可以确保点击的是正文输入区域，避免误触标题输入框
  - 不要在标题输入框中继续输入正文内容

### 错误处理
- 如果内容超出约束，暂停并要求用户修改
- 如果找不到UI元素，向用户描述当前屏幕以获取指导
- 如果发布失败，捕获错误信息并报告给用户

### 验证步骤
- 每个主要步骤后，验证操作已成功完成
- 最终发布前，可选择向用户确认："内容已准备就绪，确认发布吗？"
- 发布后，确认成功消息出现

## 使用示例

### 示例1：普通笔记
**用户请求**："帮我在小红书发布一条笔记：今天天气真好，适合出去走走。心情也跟着变好了！"

**执行流程**：
1. 识别为普通笔记
2. 打开小红书 → 红色加号 → "写文字"
3. 在"写想法"中输入简洁预览（十字以内）："今天天气真好"
4. 选择卡片风格（可询问用户喜欢哪种）→ 下一步
5. 添加标题："好天气好心情"
6. **重新输入完整正文**："今天天气真好，适合出去走走。心情也跟着变好了！"（覆盖默认的"今天天气真好"）
7. 发布笔记

### 示例2：长文笔记
**用户请求**："发布一篇长文分享我的旅行经历，标题是《三天两夜杭州游记》，内容包括景点推荐、美食打卡、住宿体验等，大概500字。添加'杭州旅游'和'旅行攻略'标签。"

**执行流程**：
1. 识别为长文笔记（内容充实）
2. 打开小红书 → 红色加号 → "写文字" → "写长文"
3. 输入标题："三天两夜杭州游记"
4. 输入详细的500字旅行内容
5. 应用"一键排版"并选择布局
6. 添加正文摘要（与图片内容不同）："杭州三日游全攻略！景点+美食+住宿一篇搞定，建议收藏！#杭州旅游 #旅行攻略"（包含用户要求的标签）
7. 发布笔记

## 常见问题排查

### 问题：普通笔记"写想法"预览内容过长
**解决方案**：
- "写想法"预览必须十字以内
- 提醒用户这只是首页展示的预览内容
- 完整内容将在后续步骤的正文中输入
- 如果用户想要发布长内容，引导其精简预览，完整内容稍后输入

### 问题：普通笔记正文只显示了"写想法"的短内容
**解决方案**：
- 这是因为忘记在最终发布界面重新输入完整正文
- **关键步骤**：在添加标题后，必须定位正文输入框
- 覆盖默认显示的"写想法"短内容
- 输入用户真正想要发布的完整内容

### 问题：长文内容过短
**解决方案**：要求用户将内容扩充至至少100字，或切换为普通笔记格式

### 问题：找不到UI元素
**解决方案**：验证小红书APP版本是否最新；不同版本UI元素名称可能有差异

### 问题：正文摘要与图片内容重复
**解决方案**：提醒用户正文应该补充而非重复图片内容。建议：添加亮点、行动号召或额外背景信息。如果用户明确要求，可以添加话题标签

## 注意事项

1. **内容质量**：确保用户提供的内容符合小红书社区规范
2. **图片vs文字**：长文中，图片展示的文字应该更详细具体，正文摘要起辅助作用
3. **标题优化**：标题应包含关键词，提高内容被发现的概率
4. **话题标签**：仅在用户明确要求添加标签时才添加。不要主动建议或添加话题标签（#）
5. **发布时机**：可建议用户在合适的时间发布以获得更好的互动效果
